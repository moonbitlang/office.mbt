# Markdown Fixture Sources

These fixtures are small acceptance inputs for `bobzhang/pdflite/markdown`.
They are generated locally with Pandoc and checked in so MoonBit tests do not
depend on network access, local fonts, or a local Pandoc installation.

## Files

- `pandoc_latin.md` and `pandoc_latin.pdf`: Latin text with an explicit page
  break, generated with Pandoc 3.8.3 and Tectonic.
- `pandoc_cjk.md` and `pandoc_cjk.pdf`: Chinese, Japanese, and Korean text,
  generated with Pandoc 3.8.3, Tectonic, and `Arial Unicode MS` on macOS.

The source Markdown files are part of this repository and covered by the
repository Apache-2.0 license. The generated PDFs are derived from those source
files and are intended only as small deterministic parser/extraction fixtures.

## Regeneration

Run:

```sh
./markdown/fixtures/generate.sh
```

The CJK fixture defaults to `Arial Unicode MS` because it provides Chinese,
Japanese, and Korean coverage with ToUnicode data in the local Tectonic path.
Override it with:

```sh
PDFLITE_PANDOC_CJK_FONT="Your CJK Font" ./markdown/fixtures/generate.sh
```

The generated bytes may vary by Pandoc, Tectonic, and font version. Review any
PDF byte diffs manually; the MoonBit tests assert extraction behavior rather
than byte identity.
